What do we do?

The lifecycle of a multitrack is as follows:

  1. Request to record a multitrack
  2. Request is approved and recording session is scheduled
  3. Recording session occurs
  4. Track gets mixed
  5. Track gets exported
  6. Track gets run through multitrack app
  7. Track is added to database
In the process of transforming a dry recording from the studio into a multitrack, things like requests to record or individual multitracks can easily be lost through miscommunication or long chains of emails. For example, a multitrack may be recorded but not mixed, or mixed but not exported.

This website is a manager for all multitracks that are recorded in Dolan and creates a "ticket" for each session to make it easier to view the status and information about a multitrack. Information includes details about the individual session and the people involved, as well as the individual multitracks recorded. Users are able to request a recording session, create a new "ticket", and view all active tickets.
