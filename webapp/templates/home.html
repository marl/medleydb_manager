MedleyDB Manager: Make Tracking Multitracks Easy

The otherwise messy task of collecting and managing multitracks is simplified by MedleyDB Manager! This collaborative website creates a "ticket" for each recording session.
The lifecycle of a multitrack is as follows:

  1. Request to record a multitrack
  2. Request is approved and recording session is scheduled
  3. Recording session occurs
  4. Track gets mixed
  5. Track gets exported
  6. Track gets run through multitrack app
  7. Track is added to database
Each ticket includes details about the recording session, the individual multitracks recorded in the session, as well as the people involved at each stage. Users can request a recording session or create a new ticket depending on the status of the multitracks. Existing tickets can be viewed, edited and kept track of on a consolidated table. Appropriate people can be contacted if questions or problems arise.
